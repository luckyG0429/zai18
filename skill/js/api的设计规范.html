1.命名

良好的一个命名习惯，就是效率开发的第一步。如果命名规范，对于自身而言，文件整理有很大的帮助，后期修改文件，可以快速的定位文件，也会显得自己很专业。 对团队而言，减少学习成本和沟通成本。

注意单复数
所有涉及遍历，操作对象，数组，集合的函数，建议都采用复数。
用词准确
1.单词含义正确；2.正反义单词用错

2.参数

对于函数而言，参数是用户设置最频繁，也是关心的部分，合理设计函数的参数，直接影响到函数的使用。

const入参
规则引入的初衷：解决函数副作用问题，如果参数是一个引用类型的数据，在函数内进行参数的修改，会导致原本的数据发生改变，往往难以查到
控制参数数量
参数的数量，超过3个，请使用对象进行封装。
前置相关性高的参数

3.作用

支持批量处理
多态处理
比如，获取和设置用同一个API
可扩展性
可扩展性，就是建议遵守开放-封闭原则。对扩展开发，对修改关闭。
避免副作用
这个，只能靠自己的写代码习惯。 1.函数体内可以使用的参数，进行操作，但是不能修改。如果修改，用一个临时变量记录参数（如果是引用类型，需要用深拷贝），这样可以避免直接修改参数 2.对于函数外的变量。比如：全局变量。函数内可以访问，但是不能被修改 3.如果需要给函数外的变量赋值，不能在函数内操作。把值返回到外部，在外进行赋值
